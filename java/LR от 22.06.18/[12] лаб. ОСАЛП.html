В этой лабораторной работе будет рассмотрен способ организации и упрощения кода программы с помощью методов. В первых заданиях вам нужно будет переписать с их помощью готовые решения задач из прошлых лабораторных. Об отличиях методов высокого качества от методов низкого качества можно прочитать в главе 7 "Высококачественные методы" части II книги "Совершенный код". (открыть в новой вкладке)

Подготовка проекта

Создайте следующую структуру проекта:
java +-.idea +-out +-src +-test
  • Отметьте папку src как "Sources Root".
  • Отметьте папку test как "Test Sources Root".
  • В папке src создайте класс Lab01. В папке test создайте класс Lab01Suite.

Задача

Реализуйте метод для решения задачи ЛР1#1860 (открыть в новой вкладке)

  • В классе Lab01 реализуйте публичный статический метод task1860.
  • Он принимает в качестве аргумента 1 строку.
  • В теле метода task1860 нужно напечатать в консоль решение задачи, используя значение аргумента в качестве исходных данных.

Проверьте корректность работы метода с помощью вспомогательной программы

  • В папке с кодом создайте класс Runner1860. В этом классе создайте метод main.
  • В методе main класса Runner1860 вызовите 3 раза метод task1860 класса Lab01.
  • При каждом вызове метода task1860 в качестве аргументов нужно передавать значения из разных тестов, описанных в задаче.
  • В результате запуска метода main класса Runner1860, в консоли должен появиться текст:
Мы стремимся к знаниям Мы стремимся к цели Мы стремимся к результату

Задача

Реализуйте метод для решения задачи ЛР2#4411 (открыть в новой вкладке)

  • В классе Lab02 реализуйте публичный статический метод task4411.
  • Он принимает в качестве аргумента 1 действительное число и возвращает действительное число.
  • В теле метода нужно вернуть результат решения задачи, используя значение аргумента в качестве исходных данных.

Проверьте корректность работы метода с помощью юнит-теста

  • В классе Lab02Suite создайте публичный метод task4411test (далее "юнит-тест"). Он помечен аннотацией @Test, ничего не возвращает и не принимает аргументы.
  • В теле юнит-теста нужно 3 раза вызвать статический метод task4411 класса Lab02. Каждый раз в качестве аргументов нужно передавать значения из разных тестов, описанных в задаче.
  • Корректность результатов каждого вызова метода task4411 нужно проверить с помощью статического метода assertEquals класса Assert.
  • Первый аргумент метода assertEquals это ожидаемое значение. Второй аргумент - результат вызова метода task4411. При проверке действительных чисел указывается третий аргумент - точность, равная 0.001
  • После запуска юнит-теста, он должен быть отмечен зеленым знаком в списке методов.
